1.чтобы отказатся от незафиксированных изменений используем команду git checkout.
2.вернуть фийл из индекса можно командой git restore --staged "file name"
3.Используем комнду git reset чтобы отменить комит в случае если комит уже запушен используем команду git revert чтобы создать копию коммита затем рекомендуется создать исправленный коммит и запушить оба этих коммита в удалённый репозиторий
4.Для создания новой ветки нужна команда git branch "имя ветки" или git chekout -b "имя ветки"
5.разница между fast-forward merge  и non fast-forfard merge заключается в том что во втором случае гит неможет сам осуществить слияние истории веток
6.конфликт слияния может возникнуть если в разных веткках были разные модификации одних и тех же файлов в таком случае пользователь должен вручную указать гиту какие изменения следует сохранить при слиянии
7.да но только в том случае если вветке нет незакомичиных изменений и пользователь не находится в этой ветке для этого нужна команда git branch -d "branchname"
